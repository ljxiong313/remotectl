# RemoteCtl 使用手册

## 一、产品简介

RemoteCtl 是一款开源的跨平台远程桌面控制工具，基于 WebRTC 技术实现端到端的视频流传输。您可以通过浏览器或桌面客户端远程控制另一台计算机。

### 主要特性

- 🌐 **浏览器控制**：无需安装软件，通过浏览器即可远程控制
- 🔒 **端到端加密**：所有数据传输采用 E2EE 加密
- ⚡ **低延迟**：P2P 直连传输，局域网延迟低于 50ms
- 🖥️ **跨平台**：支持 Windows、macOS、Linux、Android、iOS
- 🏠 **私有部署**：支持自建服务器，数据完全自主可控
- 💬 **会话聊天**：支持远程会话内的文字聊天和文件传输

---

## 二、快速开始

### 2.1 控制端（主动控制方）

#### 方式一：浏览器访问（推荐）

1. 打开浏览器，访问您的 RemoteCtl 服务器地址
2. 进入「远程控制」模式
3. 输入被控端的设备 ID 和会话密码
4. 点击「连接」按钮，等待连接建立

#### 方式二：桌面客户端

1. 下载对应平台的桌面客户端
2. 安装并启动应用
3. 进入「远程控制」模式
4. 输入被控端的设备 ID 和会话密码
5. 点击「连接」按钮

### 2.2 被控端（被控制的电脑）

1. 下载并安装 RemoteCtl Agent
2. 运行 Agent 程序
3. 首次运行会显示设备 ID 和会话密码
4. 将设备 ID 和会话密码告知控制端
5. 等待控制端发起连接请求

## 三、功能详解

### 3.1 远程桌面控制

连接成功后，您可以：

- **鼠标控制**：移动鼠标、点击、拖拽
- **键盘输入**：输入文字、快捷键组合
- **剪贴板同步**：复制粘贴跨设备生效
- **滚动控制**：鼠标滚轮操作

### 3.2 屏幕共享模式

被控端可以主动共享自己的屏幕给控制端观看（仅观看，不可控制）：

1. 被控端进入「共享此电脑」模式
2. 系统会生成一个共享链接和密码
3. 控制端通过链接访问，仅能观看屏幕
4. 被控端可随时结束共享

### 3.3 虚拟键盘（移动端）

使用手机或平板控制时，虚拟键盘提供完整的输入能力：

- **功能键**：Ctrl、Shift、Alt、Win/Cmd 可单独按下
- **数字行**：F1-F12 快捷键
- **方向键**：上下左右导航
- **组合键**：支持 Ctrl+B+1 等组合操作

### 3.4 会话聊天

在远程会话过程中，双方可以发送文字消息：

1. 点击工具栏的聊天图标打开聊天面板
2. 输入消息并发送，对方会收到通知
3. 支持发送链接、简短文字
4. 聊天消息不经过服务器，直接 P2P 传输

### 3.5 文件传输

远程会话中可以双向传输文件：

- **控制端 → 被控端**：发送文件到被控端指定目录
- **被控端 → 控制端**：请求获取被控端文件
- 文件直接 P2P 传输，不经过服务器
- 支持断点续传，大文件自动分块传输

## 四、画质与性能设置

### 4.1 画质参数说明

| 参数 | 说明 | 建议值 |
|------|------|--------|
| 码率 | 视频编码的比特率 | 局域网: 8Mbps, 4G: 1-2Mbps |
| 帧率 | 每秒传输的画面数量 | 局域网: 30-60fps, 移动网络: 15-30fps |
| 缩放比例 | 相对于原分辨率的缩放 | 高速网络: 1.0, 低速网络: 0.5 |

### 4.2 推荐配置

**局域网环境**：
- 码率：4-8 Mbps
- 帧率：30-60 fps
- 缩放：1.0（原分辨率）

**4G/5G 移动网络**：
- 码率：1-2 Mbps
- 帧率：15-30 fps
- 缩放：0.5（半分辨率）

**弱网环境**：
- 码率：500k-1 Mbps
- 帧率：10-15 fps
- 缩放：0.25（四分之一分辨率）

## 六、常见问题

### 6.1 连接问题

**Q: 连接失败，显示「连接超时」？**

A: 请检查：
1. 被控端是否在线并运行 RemoteCtl Agent
2. 设备 ID 和会话密码是否正确
3. 防火墙是否允许 443 端口
4. 尝试使用 TURN 中继（移动网络必需）

**Q: 提示「连接被拒绝」？**

A: 请检查：
1. 确认被控端已接受连接请求
2. 检查会话密码是否正确
3. 尝试重新启动被控端 Agent

### 6.2 画面问题

**Q: 画面延迟很高？**

A: 建议：
1. 优先使用局域网连接
2. 降低画质设置（码率、帧率、缩放）
3. 检查网络带宽是否充足
4. 关闭其他占用网络的程序

**Q: 画面卡顿或掉帧？**

A: 建议：
1. 降低帧率设置
2. 检查被控端 CPU/GPU 占用
3. 关闭被控端不必要的程序

### 6.3 控制问题

**Q: 鼠标点击位置不准？**

A: 建议：
1. 检查分辨率缩放设置是否一致
2. 尝试调整画质设置中的缩放比例
3. 重启被控端 Agent

**Q: 键盘输入无法识别？**

A: 建议：
1. 使用虚拟键盘输入特殊字符
2. 检查输入法状态，切换到英文输入
3. 尝试 Ctrl+Space 切换输入法

## 七、安全说明

### 7.1 数据加密

- 视频流和控制指令采用端到端加密（E2EE）
- 信令传输使用 TLS 加密
- 服务器不存储任何明文数据
- 每次会话使用一次性密钥

### 7.2 安全建议

- 会话密码仅使用一次，建议每次重新获取
- 不要在公共网络下进行敏感操作
- 定期更新 RemoteCtl 到最新版本
- 启用防火墙，限制不必要的端口访问

### 7.3 隐私保护

- 服务器仅转发信令，不接触视频数据
- 聊天消息直接 P2P 传输，不经过服务器
- 文件传输端到端加密，服务器无法解密
- 会话结束后，所有临时数据自动清除

---

## 八、卸载与清理

### 8.1 卸载 Agent

**Windows**：
1. 关闭 RemoteCtl Agent 程序
2. 在控制面板 → 程序和功能 中卸载
3. 或删除程序所在文件夹

**macOS/Linux**：
1. 关闭 RemoteCtl Agent 程序
2. 删除程序文件夹
3. 清理配置文件：`rm -rf ~/.config/remotectl`
