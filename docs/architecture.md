# RemoteCtl 架构设计文档

## 一、系统架构概述

RemoteCtl 采用分层架构设计，将系统划分为四个核心组件：

```
┌─────────────────────────────────────────────────────────────────────┐
│                      Web 浏览器 / 桌面客户端                        │
│     ┌─────────────┐  ┌─────────────┐  ┌─────────────┐            │
│     │  控制界面   │  │ WebRTC模块 │  │  信令模块  │            │
│     └─────────────┘  └─────────────┘  └─────────────┘            │
└─────────────────────────────────────────────────────────────────────┘
                                    │
          ┌────────────────────────┴────────────────────────┐
          │                 WebRTC P2P 通道                  │
          │   视频流(加密) │ 控制指令(加密) │ 文件数据     │
          └────────────────────────┬────────────────────────┘
                                    │
┌─────────────────────────────────────────────────────────────────────┐
│                          被控端 (Agent)                            │
│     ┌─────────────┐  ┌─────────────┐  ┌─────────────┐            │
│     │  屏幕捕获   │  │ 视频编码   │  │  WebRTC    │            │
│     ├─────────────┤  ├─────────────┤  ├─────────────┤            │
│     │  输入处理   │  │ 剪贴板同步 │  │  文件传输  │            │
│     └─────────────┘  └─────────────┘  └─────────────┘            │
└─────────────────────────────────────────────────────────────────────┘
                                    │
          ┌────────────────────────┴────────────────────────┐
          │                   信令服务器 (Server)               │
          │   ┌─────────────┐  ┌─────────────┐  ┌─────────────┐│
          │   │   信令转发  │  │  设备注册   │  │   会话管理  ││
          │   └─────────────┘  └─────────────┘  └─────────────┘│
          └────────────────────────┬────────────────────────┘
                                    │
          ┌────────────────────────┴────────────────────────┐
          │               TURN 中继服务器 (coturn)            │
          └─────────────────────────────────────────────────┘
```

## 二、组件说明

### 2.1 被控端 Agent

**技术栈**：Go + C/C++（平台特定实现）

**核心模块**：

| 模块 | 功能 | 依赖 |
|------|------|------|
| capture | 屏幕捕获，支持多显示器 | GDI/X11/Cocoa |
| input | 键盘鼠标输入模拟 | WinAPI/Carbon/X11 |
| pipeline | 视频编码和WebRTC流管理 | libx264/pion-webrtc |
| session | 会话生命周期管理、认证 | TLS/ECDHE |

**目录结构**：

```
agent/
├── capture/          # 屏幕捕获模块
│   ├── screen.go     # 屏幕信息获取
│   └── x11.go        # X11屏幕捕获(Linux)
├── input/            # 输入控制模块
│   ├── keyboard.go   # 键盘事件处理
│   └── mouse.go      # 鼠标事件处理
├── pipeline/         # 数据管道模块
│   ├── encoder.go    # 视频编码器封装
│   └── rtc.go        # WebRTC连接管理
├── session/          # 会话管理模块
│   ├── manager.go    # 会话管理器
│   └── crypt.go      # 加密处理
├── platform_*.go     # 平台特定实现
└── main.go           # 程序入口
```

### 2.2 信令服务器 Server

**技术栈**：Go

**核心功能**：
- 设备注册和发现
- WebRTC 信令转发（SDP/ICE）
- 会话管理和认证
- 不处理任何视频/音频数据

**配置示例**：

```yaml
server:
  addr: ":443"
  cert: "./certs/cert.pem"
  key: "./certs/key.pem"
turn:
  url: "turn:your-server:3478"
  user: "remotectl"
  password: "your-password"
```

### 2.3 Web 客户端 Client

**技术栈**：Vue3 + TypeScript + WebRTC

**核心组件**：

| 组件 | 功能 |
|------|------|
| RemoteView.vue | 远程桌面显示 |
| ControlBar.vue | 控制工具栏 |
| VirtualKB.vue | 虚拟键盘（移动端） |
| ChatPanel.vue | 聊天面板 |

**核心 Composable**：

- `useWebRTC.ts` - WebRTC 连接封装
- `useSignaling.ts` - WebSocket 信令连接
- `useInput.ts` - 输入事件处理

### 2.4 桌面应用 App

**技术栈**：Flutter + Dart

**功能模式**：
- 远程控制模式：控制其他设备
- 共享本机模式：被其他设备控制

## 三、数据流设计

### 3.1 视频流

```
屏幕帧 → 编码器(H.264) → WebRTC VideoTrack → P2P加密传输 → 解码 → 显示
```

**特点**：
- P2P 直连传输，服务器不接触视频数据
- 支持 H.264 硬件编码
- 可调节码率、帧率、分辨率

### 3.2 控制指令流

```
输入事件 → 加密(ECDH+AES-256-GCM) → DataChannel → 解密 → 输入模拟
```

**特点**：
- 端到端加密，服务器无法解密
- 低延迟本地光标反馈
- 支持键盘组合键

### 3.3 信令流

| 消息类型 | 方向 | 说明 |
|----------|------|------|
| register | Agent→Server | 设备注册 |
| connect | Client→Server | 发起连接 |
| offer/answer | 双向 | SDP交换 |
| ice-candidate | 双向 | ICE候选交换 |
| disconnect | 双向→Server | 断开连接 |

## 四、安全设计

### 4.1 端到端加密（E2EE）

- **密钥交换**：ECDH P-256
- **数据加密**：AES-256-GCM
- **会话隔离**：每次会话生成新密钥对

### 4.2 传输安全

- 信令传输：TLS 1.2+
- TURN 认证：lt-cred-mech
- 证书验证：防止中间人攻击

### 4.3 会话认证

- 随机生成 8 位数字密码
- 每次会话独立密码
- 密码显示在悬浮球/界面

```
┌────────┐         ┌─────────┐         ┌─────────┐
│ Agent  │         │ Server  │         │ Client  │
└───┬────┘         └───┬─────┘         └───┬─────┘
    │                  │                   │
    │──register───────>│                   │
    │<─────ack─────────│                   │
    │                  │<────connect──────│
    │<─────offer───────│<───offer─────────│
    │────answer───────>│────answer───────>│
    │<──ice-candidate──│<──ice-candidate──│
    │                  │                   │
    │═════════════════════════════════════│
    │         (P2P 视频流/控制指令)        │
    │═════════════════════════════════════│
```

## 六、NAT 穿透策略

| 优先级 | 方式 | 说明 |
|--------|------|------|
| 1 | Host Candidate | 局域网直连 |
| 2 | Srflx Candidate | STUN 发现公网 IP |
| 3 | Relay Candidate | TURN 中继 |
